## 1. 프로젝트 개요

이 저장소는 외교공공데이터 활용 공모전 프로젝트에서 사용하는 **재외공관 민원 RAG 데이터셋**입니다.

해외에 체류하는 국민이 여권, 재외국민등록, 공증·영사확인, 가족관계, 병역, 사건사고 등 재외공관 민원을 처리할 때 필요한 절차와 서류를 AI가 검색하고 안내할 수 있도록 데이터를 JSON 및 JSONL 형태로 구조화했습니다.

이 데이터는 민원유형 분류 Agent, RAG 검색, 신청서·사유서 초안 생성, Judge LLM 검증 로직에 연결하기 위한 기반 데이터입니다.

## 5. RAG JSONL 구조

`재외공관_민원_사건사고_RAG_JSONL_카테고리통일.jsonl`은 한 줄이 하나의 검색 청크입니다.

각 줄은 대략 아래 구조를 가집니다.

~~~json
{
  "chunk_id": "청크 고유 ID",
  "record_id": "원본 레코드 ID",
  "data_type": "civil_service 또는 emergency_manual",
  "chunk_type": "full_record",
  "content": "RAG 검색 대상 본문",
  "metadata": {
    "category": "통일 카테고리",
    "original_category": "원래 카테고리",
    "category_normalized": "통일 카테고리",
    "country": "국가",
    "mission": "공관",
    "title": "민원명 또는 사건사고 유형",
    "keywords": [],
    "source_trace": {},
    "example_questions": []
  }
}
~~~

### data_type 설명

| data_type | 의미 |
|---|---|
| `civil_service` | 국가·공관별 일반 민원 데이터 |
| `emergency_manual` | 사건사고 대응 매뉴얼 데이터 |

## 6. 통합 JSON 설명

`재외공관_민원_사건사고_통합_JSON_코퍼스.json`은 JSONL로 변환하기 전의 통합 원본입니다.

주요 구조는 다음과 같습니다.

~~~json
{
  "services": "국가·공관별 일반 민원",
  "emergency_manuals": "사건사고 대응 매뉴얼",
  "qa_pairs": "전체 QA 쌍",
  "source_audit": "수집 및 파싱 검증 정보"
}
~~~

이 파일은 실제 RAG 검색보다는 원본 구조 확인, 디버깅, 데이터 검수용으로 사용하면 됩니다.

## 7. 테스트 질문 세트 설명

`재외공관_민원_RAG_테스트질문세트.json`은 AI 로직 테스트용 정답지입니다.

각 테스트 케이스에는 다음 정보가 포함되어 있습니다.

- 사용자 질문
- 기대 카테고리
- 기대 국가
- 기대 공관
- 기대 민원명 또는 사건사고 유형
- RAG 대상 `record_id`
- 초안 생성 필요 여부
- 필요한 추가 질문
- 검색 결과에 포함되면 좋은 키워드

예시:

~~~json
{
  "user_query": "미국 시애틀에서 여권을 잃어버렸고 다음 주에 한국에 가야 해. 뭘 준비해야 해?",
  "expected": {
    "route": "civil_service",
    "category": "여권",
    "country": "미국",
    "mission": "주시애틀대한민국 총영사관",
    "title": "여권 분실 신고",
    "needs_draft": true,
    "draft_template": "여권분실 경위서"
  }
}
~~~

## 8. AI 로직 담당자 사용 순서

### 1단계: RAG 데이터 적재

`재외공관_민원_사건사고_RAG_JSONL_카테고리통일.jsonl`을 벡터DB에 적재합니다.

권장 필터링 필드:

- `metadata.category`
- `metadata.country`
- `metadata.mission`
- `metadata.title`
- `data_type`

### 2단계: 민원유형 분류 Agent 테스트

`재외공관_민원_RAG_테스트질문세트.json`의 `user_query`를 입력으로 넣고, AI가 아래 값을 맞히는지 확인합니다.

- category
- country
- mission
- title
- route
- needs_draft

### 3단계: RAG 검색 테스트

분류 결과를 기준으로 JSONL에서 관련 청크가 검색되는지 확인합니다.

예를 들어 사용자가 “미국 시애틀에서 여권을 잃어버렸다”고 하면:

- `category = 여권`
- `country = 미국`
- `mission = 주시애틀대한민국 총영사관`
- `title = 여권 분실 신고`

조건에 맞는 청크가 상위 검색 결과로 나와야 합니다.

### 4단계: Judge LLM 검증

생성된 답변이 RAG 청크의 `content`와 `metadata.source_trace`에 근거하는지 확인합니다.

원문에 없는 내용은 임의로 생성하지 않고, “공관 확인 필요”로 표시해야 합니다.

## 9. 신청서·초안 생성과 연결

테스트 질문 세트의 `expected.needs_draft`와 `expected.draft_template`을 사용하면 신청서 초안 생성 로직과 연결할 수 있습니다.

예시 템플릿:

| draft_template | 사용 상황 |
|---|---|
| `여권분실 경위서` | 여권 분실 신고 |
| `공증 사유서` | 위임장, 진술서, 확인서 인증 |
| `출생신고 초안` | 해외 출생신고 |
| `병역 국외여행 기간연장 사유서` | 병역의무자 국외여행 기간연장 |

## 10. 주의사항

이 데이터는 전체 외교부 민원 데이터를 모두 수집했다는 의미가 아닙니다.  
프로젝트에서 정한 범위 안에서 수집·정제·구조화한 RAG용 코퍼스입니다.

현재 포함 범위는 다음과 같습니다.

- 미국
- 일본
- 중국
- 베트남
- 캐나다
- 사건사고 대응 매뉴얼

서비스 기획에는 비자 민원도 포함되어 있지만, 현재 업로드된 코퍼스에는 별도 비자 레코드가 포함되어 있지 않습니다.

또한 원문에 없는 구비서류, 수수료, 법률 해석은 임의로 보완하지 않았습니다.  
최신 정보 확인이 필요한 경우 실제 서비스에서는 “공관 확인 필요”로 표시해야 합니다.

## (추가) 정형 데이터 (외교부 공공데이터 API) 연동 내역

이 프로젝트는 RAG 기반의 비정형 문서 검색(민원 절차, 서류 안내)에만 그치지 않고, 사용자에게 정확한 위치 및 실시간 안전 정보를 제공하기 위해 아래 4종의 외교부 정형 데이터(JSON)를 서비스 UI와 결합하여 사용합니다.

### 연동된 정형 데이터 파일 및 UI 적용 방안

1. `embassy.json` (국가·지역별 재외공관 정보)
   - **용도:** 사용자의 체류 지역과 가장 가까운 관할 재외공관 탐색
   - **UI 적용:** 주소, 전화번호, 지도 렌더링용 위·경도 및 **업무시간(점심시간 포함)** 안내로 헛걸음 방지

2. `embassy_homepage.json` (재외공관 홈페이지 정보)
   - **용도:** 공관별 공식 웹사이트 연결
   - **UI 적용:** 민원 리포트 하단에 '관할 공관 공식 홈페이지 바로가기' 버튼 제공

3. `safety_notice.json` (국가·지역별 안전공지)
   - **용도:** 체류 국가의 최신 사건사고 및 안전 유의사항 파악
   - **UI 적용:** 긴급 상황(예: 태풍, 파업, 소요사태 등) 발생 시 결과 리포트 최상단에 실시간 경고 배너 노출

4. `travel_alert.json` (국가·지역별 여행경보 조정)
   - **용도:** 체류 중인 국가의 전반적인 위험도 안내
   - **UI 적용:** 국가 검색 시 해당 지역의 여행경보 단계(남색/황색/적색/흑색)를 시각적인 배지로 제공

**💡 서비스 통합 시너지 (AI + 정형 데이터):**
AI 분류 Agent가 비정형 데이터(RAG)에서 '민원 서류 초안'과 '절차'를 생성하면, 최종적으로 이 정형 데이터들과 결합합니다. 결과적으로 사용자는 **[작성된 서류 초안 + 방문할 공관의 영업시간 + 해당 지역 실시간 안전 경보]**가 하나로 통합된 맞춤형 민원 패키지를 제공받게 됩니다.
